### Описание сервисов
Службы Analysis Services — это подсистема аналитических данных (VertiPaq), используемая для поддержки принятия решений и бизнес-аналитики.

1. Power BI — подсистема служб Analysis Services поддерживает артефакт семантической модели (ранее известный как набор данных) в Power BI. Семантические модели создаются в Power BI Desktop, а затем публикуются в служба Power BI.
2. Microsoft Fabric — Microsoft Fabric включает Power BI. Как и Power BI, подсистема Analysis Services поддерживает семантические модели Microsoft Fabric.
3. SQL Server Analysis Services — устанавливается как локальный экземпляр или экземпляр сервера виртуальной машины, SQL Server Analysis Services поддерживает табличные модели на всех уровнях совместимости (в зависимости от версии), многомерных моделей и PowerPivot для SharePoint.
4. Analysis Services Tabular - это технология анализа данных, которая позволяет организациям быстро извлекать информацию из больших объемов данных. Она использует модель данных на основе таблиц, которая обеспечивает высокую скорость обработки запросов и гибкость в работе с данными. Tabular модели позволяют создавать сложные аналитические запросы и модели, а также легко интегрироваться с другими инструментами бизнес-аналитики.

Azure Analysis Services — это полностью управляемая платформа как услуга (PaaS), которая предоставляет модели данных корпоративного уровня в облаке. С помощью расширенных функций комбинирования и моделирования можно объединять данные из нескольких источников данных, определять метрики и защищать данные в одной доверенной семантической табличной модели данных.

1. Режим **In-Memory** является табличной моделью по умолчанию и поддерживает несколько источников данных.
2. Режим DirectQuery использует базовую реляционную базу данных для хранения и выполнения запросов. Поддерживаются чрезвычайно большие наборы данных в одном SQL Server, хранилище данных SQL Server, база данных Azure SQL Database, Azure SQL Data Warehouse, Oracle и Teradata.
3. Azure Analysis Services Basic - это базовое предложение службы анализа данных в облаке Azure. Это инструмент бизнес-аналитики, который позволяет анализировать и визуализировать данные в реальном времени. Он предоставляет возможности интеллектуального анализа данных, моделирования и отчетности, а также поддерживает несколько источников данных. Basic edition предлагает ограниченные ресурсы и возможности по сравнению с другими планами. Может быть подходящим для небольших рабочих нагрузок или для тех, кто хочет попробовать службу перед обновлением до более продвинутого плана.
4. Azure Analysis Services Developer - это уровень подписки, который предоставляет полный доступ ко всем функциям и возможностям службы анализа данных. Он включает в себя все возможности Basic-версии, а также дополнительные возможности для разработки и тестирования, такие как поддержка многопользовательской разработки, создание и отладка моделей и расширенные возможности диагностики. 
5. Azure Analysis Services Standard - это уровень подписки, предлагающий расширенные функции и повышенную производительность для сложных рабочих нагрузок аналитики. Он включает все возможности уровня Basic, а также увеличенные ресурсы, поддержку высокой доступности и возможность масштабирования для больших объемов данных. Может быть подходящим выбором для средних и крупных организаций, которым требуется стабильная производительность и масштабируемость для своих рабочих нагрузок бизнес-аналитики.

База данных Azure для PostgreSQL — это служба реляционных баз данных на основе СУБД PostgreSQL с открытым исходным кодом. Это полностью управляемая база данных как услуга, которая может обслуживать критически важные рабочие нагрузки, обеспечивая прогнозируемую производительность и безопасность, а также высокий уровень доступности и динамическое масштабирование. База данных Azure для PostgreSQL на базе выпуска сообщества PostgreSQL доступен в двух режимах развертывания:

1. Гибкий сервер Базы данных Azure для PostgreSQL — это полностью управляемая служба базы данных, которая обеспечивает более детализированный контроль и гибкость функций управления базами данных, а также параметрами конфигурации.
2. Отдельный сервер — это полностью управляемая служба базы данных с минимальными требованиями к настройке базы данных. Платформа отдельного сервера рассчитана на обработку большинства функций управления базами данных, как, например, установка исправлений, резервное копирование, обеспечение высокого уровня доступности и безопасности с минимальным вмешательством пользователя дли их настройки и управления. Эта архитектура оптимизирована для обеспечения уровня доступности 99,99 % при использовании одной зоны доступности.

Azure Databricks — это единая, открытая платформа аналитики для создания, развертывания, совместного использования и обслуживания корпоративных данных, аналитики и решений искусственного интеллекта в масштабе.
  1. Azure Databricks для Data Analytics - это облачное решение от Microsoft, которое позволяет организациям анализировать большие объемы данных с использованием Apache Spark. Оно предоставляет масштабируемую и гибкую платформу для обработки и анализа данных, а также возможность интеграции с другими сервисами Azure, такими как Blob Storage, SQL Database и Machine Learning. Позволяет создавать аналитические модели и прогнозы, а также визуализировать результаты с помощью встроенных инструментов.
  2. Azure Databricks for Data Engineering - это решение для работы с данными, которое помогает реобразовывать данные в информацию. Оно включает в себя инструменты для очистки данных, их интеграции и преобразования, а также возможности машинного обучения и искусственного интеллекта. С помощью него можно создавать конвейеры данных, управлять ими и оптимизировать их производительность. Это позволяет быстрее и эффективнее обрабатывать большие объемы данных.

Azure Monitor — это комплексное решение для мониторинга для сбора, анализа и реагирования на данные мониторинга из облачных и локальных сред. Можно использовать для повышения доступности и производительности приложений и служб.
  1. Azure Monitor Notifications - это функция, которая позволяет автоматизировать отправку уведомлений о важных событиях и состоянии ресурсов в облачной среде. С помощью этой функции можно настроить отправку уведомлений по электронной почте, SMS или push-уведомлений в приложении, когда происходят определенные события, такие как : ошибки в работе приложений, превышение пороговых значений показателей производительности и другие изменения в состоянии ресурсов.
  2. Azure Monitor Notifications Webhook - это механизм, который позволяет автоматически вызывать HTTP-запрос при наступлении определенного события.
  3. Azure Monitor Notifications Email - это функция, позволяющая автоматически отправлять уведомления о различных событиях в облачной платформе Microsoft Azure на указанный адрес электронной почты.

Azure  Virtual Machines — один из нескольких типов запрашиваемых масштабируемых вычислительных ресурсов, которые предоставляет Azure. Обычно виртуальную машину выбирают, когда требуется более строгий контроль за вычислительной средой, чем в других вариантах.
  1. Azure Virtual Machines Series Basic -  - это базовые виртуальные машины, в рамках службы Azure. Они представляют собой недорогие, экономичные решения для разработчиков и предприятий. Серия Basic предлагает различные размеры виртуальных машин, начиная от небольших и недорогих решений для разработки и заканчивая более мощными машинами для производственных сред. Однако, эти машины ограничены по некоторым функциям, таким как доступ к SSD-дискам и возможности масштабирования.
  2. Azure Virtual Machines (A-Series) - это серия виртуальных машин, предлагаемая в Azure. Они оптимизированы для приложений, требующих высокой производительности процессора. A-серия включает машины с процессорами Intel® Xeon® Scalable и AMD EPYC™ седьмого поколения, обеспечивающие производительность до 400 Вт на сокет. Эти машины подходят для рабочих нагрузок, таких как базы данных, аналитика и высокопроизводительные вычисления.
  3. Azure Virtual Machines (B-Series) - эта серия оптимизирована для рабочих нагрузок с интенсивным использованием операций ввода-вывода, таких как большие данные, интернет вещей и облачные приложения. B-серия предлагает различные размеры машин с разными комбинациями процессоров, объема оперативной памяти и пропускной способности сети. Они также поддерживают диски SSD и HDD, а также ускорители сети, что делает их идеальными для приложений с высокими требованиями к вводу-выводу.
  4. Azure Virtual Machines (C-Series) - это виртуальные машины с GPU, предназначенные для ускорения работы приложений с интенсивными вычислениями, таких как машинное обучение, графика и научные расчеты. Эти машины оснащены GPU NVIDIA Tesla, что позволяет ускорить выполнение ресурсоемких задач.
  5. Azure Virtual Machines (D-Series) являются высокопроизводительными виртуальными машинами, предназначенными для вычислительных рабочих нагрузок, например, для баз данных и аналитики. Эти машины предлагают масштабируемую производительность процессора, памяти и хранилища, что позволяет удовлетворять потребности растущих рабочих нагрузок без необходимости перенастройки или перераспределения ресурсов. Кроме того, D-серия поддерживает технологию Intel® Turbo Boost, позволяющую увеличивать производительность в зависимости от рабочей нагрузки.
  6. Azure Virtual Machines (E-series) - это  виртуальные машины, оптимизированные для вычислений на базе графических процессоров. Они оснащены графическими процессорами NVIDIA A100 Tensor Core, которые обеспечивают значительное ускорение для широкого спектра приложений, включая глубокое обучение, высокопроизводительные вычисления и симуляции. E-серия предлагает масштабируемую производительность.
  7. Azure Virtual Machines (F-series) являются виртуальными машинами с оптимизированной структурой для рабочих нагрузок SAP HANA.(*SAP HANA - это платформа управления данными и база данных в оперативной памяти, разработанная компанией SAP. Она предназначена для работы с большими объемами данных в режиме реального времени, обеспечивая высокую производительность и эффективность обработки данных.)
  8. Azure Virtual Machines Standard - это серия виртуальных машин,  которые оптимизированы для обеспечения надежной и производительной работы приложений и рабочих нагрузок. Standard-серия предлагает широкий выбор размеров машин, от небольших и экономичных до более мощных и масштабируемых. Эти машины поддерживают необходимые функции, такие как диски SSD, масштабирование, резервное копирование и восстановление, а также доступ к рынкам приложений Azure.
  9. Azure Virtual Machines (G-series) - это новый тип виртуальных машин, которые оптимизированы для GPU-ускоренной работы. Они подходят для задач, связанных с искусственным интеллектом, глубоким обучением и высокопроизводительными вычислениями. G-серия предоставляет GPU NVIDIA Tesla P40, который обеспечивает значительное ускорение работы. Кроме того, эти машины поддерживают технологию NVIDIA NVLink, которая обеспечивает более быструю передачу данных между GPU и процессором.
  10. Azure Virtual Machines (H-series) - это виртуальные машины, предназначенные для работы с базами данных и аналитическими нагрузками. Они оптимизированы для использования процессоров Intel Xeon Platinum 8168 и 64-битной архитектуры, что обеспечивает высокую производительность при выполнении сложных вычислений. Машины H-серии также имеют большой объем оперативной памяти и поддерживают диски NVMe, что позволяет работать с объёмными данными.
  11. Azure Virtual Machines (L-series) - это виртуальные машины, разработанные специально для больших наборов данных и приложений с большими рабочими нагрузками. Эти машины имеют высокую пропускную способность и большой объем доступной памяти.
  12. Azure Virtual Machines (M-series) - это серия виртуальных машин, оптимизированных для работы с графическими приложениями и играми. Они оснащены высокопроизводительными графическими процессорами, которые позволяют работать с 3D-моделями и анимацией.
  13. Azure Virtual Machines (N-series) - это новая серия виртуальных машин, предназначенных для работы с высокопроизводительными вычислительными нагрузками, такими как научные вычисления, моделирование и финансовые расчеты. Эти машины используют процессоры Intel Xeon Scalable и могут оснащаться ускорителями NVIDIA Tesla V100. N-серия также обеспечивает поддержку дисков NVMe.
  14. Azure Virtual Machines S-series - это серия виртуальных машин, разработанная специально для работы с приложениями, требующими высокой вычислительной мощности. Они оснащены процессорами Intel Xeon Scalable и NVIDIA Tesla M60, что обеспечивает увеличение производительности по сравнению с другими сериями виртуальных машин. S-серия также предоставляет возможность использовать диски SSD и NVMe, что обеспечивает более быстрое время отклика и большую пропускную способность для приложений.

Azure Event Grid   — это полностью масштабируемая полностью управляемая служба распространения сообщений Pub Sub, которая предлагает гибкие шаблоны потребления сообщений с помощью протоколов MQTT и HTTP. С помощью Сетка событий Azure можно создавать конвейеры данных с данными устройства, интегрировать приложения и создавать бессерверные архитектуры на основе событий.
  1. Operations - это раздел в панели управления Azure, который позволяет управлять различными аспектами работы Event Grid. Здесь можно создавать и настраивать подписки на события, управлять правилами маршрутизации и просматривать журналы событий.

Windows Azure - это облачный сервис Microsoft, который предлагает различные услуги, включая хранение данных, обработку данных, базы данных, аналитику и т.п
1. Windows Azure Data Transfer - это служба, которая позволяет организациям передавать данные между локальными и облачными средами.
2. DataTransfer Out - это процесс передачи данных из локальной среды в облако.
3.  Azure Storage - это облачное хранилище данных, которое позволяет хранить и извлекать данные с высокой степенью масштабируемости и доступности. Оно предлагает три типа хранилищ: Blob Storage, Table Storage и Queue Storage. Blob Storage используется для хранения неструктурированных данных, таких как файлы и изображения, Table Storage - для хранения структурированных данных в виде таблиц, а Queue Storage - для обмена сообщениями и асинхронной обработки задач.
4.  Azure SQL Database - это реляционная база данных, которая обеспечивает высокую производительность, масштабируемость и безопасность. Она поддерживает различные языки программирования и платформы разработки, такие как .NET, Java и PHP.
5.   App Services - это платформа облачных вычислений, которая позволяет создавать, развертывать и масштабировать веб-приложения и мобильные приложения. Она предоставляет такие функции, как автоматическая масштабирование, мониторинг производительности, управление безопасностью и многое другое.
6.    Active Directory - это система управления пользователями и группами, которая позволяет централизованно управлять доступом к ресурсам в облачной среде.
7.  Virtual Machines - это виртуальные машины, которые позволяют создавать и запускать собственные виртуальные машины в облачной инфраструктуре.
8.  Batch AI - это платформа для обработки больших объемов данных с использованием технологий машинного обучения.

### Описание сервисов Yandex Cloud 
Yandex DataLens — это сервис для бизнес-аналитики. Сервис позволяет подключаться к различным источникам данных, строить визуализации, собирать дашборды и делиться полученными результатами.

---

Managed Service for PostgreSQL помогает разворачивать и поддерживать кластеры серверов PostgreSQL в инфраструктуре Yandex Cloud. Сервис поддерживает PostgreSQL 11, 12, 13, 14, 15 и 16, а также PostgreSQL 11, 12, 13, 14 и 15 для 1C.

---

Yandex Monitoring позволяет собирать и хранить метрики, а также отображать их в виде графиков на дашбордах.

---

Сервис Yandex Compute Cloud предоставляет масштабируемые вычислительные мощности для создания виртуальных машин и управления ими. Сервис поддерживает прерываемые виртуальные машины, а также отказоустойчивые [г](https://cloud.yandex.ru/docs/compute/concepts/instance-groups/)руппы виртуальных машин

---

Yandex DataSphere — это сервис для ML-разработки полного цикла, который позволяет использовать все преимущества облачных технологий, сочетая в себе простой и понятный веб-интерфейс, привычные IDE и возможность бесшовного использования разных конфигураций вычислительных ресурсов Yandex Cloud.

---

Yandex Virtual Private Cloud служит для управления облачными сетями и связи облачных ресурсов между собой и с интернетом. Virtual Private Cloud позволяет создавать сети, подсети в зонах доступности Yandex Cloud, назначать облачным ресурсам внутренние и публичные IP-адреса, а также предоставляет возможность защиты от DDoS-атак с помощью компонента DDoS-Protection.

---

Yandex Network Load Balancer (NLB) предназначен для построения отказоустойчивых и масштабируемых приложений и сервисов в Yandex Cloud. Отказоустойчивость и масштабируемость достигаются за счет распределения трафика, поступающего на балансировщик, по облачным ресурсам которые размещаются за ним.

---

Yandex Message Queue — универсальное масштабируемое решение для обмена сообщениями между приложениями. Для работы с сервисом можно использовать популярные инструменты: API сервиса совместим с Amazon SQS API.

---

Yandex IoT Core — это облачный отказоустойчивый MQTT-сервер, позволяющий наладить двустороннюю защищенную связь между устройствами и локальными или облачными ресурсами.

---

Managed Service for MySQL - помогает разворачивать и поддерживать кластеры серверов MySQL в инфраструктуре Yandex Cloud. Сервис поддерживает MySQL 5.7 и MySQL 8.0.

---

Сервис Yandex Object Storage — это универсальное масштабируемое решение для хранения данных. Оно подходит как для высоконагруженных сервисов, которым требуется надежный и быстрый доступ к данным, так и для проектов с невысокими требованиями к инфраструктуре хранения.
